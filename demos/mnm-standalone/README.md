# MnM Standalone: True MnM Interpreter

MnM programs executed by `mnm_vm.arnoldc` — the true ArnoldC interpreter that reads MnM from stdin.

## Quick Start

```bash
# Compile the interpreter (once):
java -jar ../../ArnoldC-patched.jar ../../mnm_vm.arnoldc

# Factorial(5):
python3 ../../mnm_to_stdin.py factorial.mnm factorial.mnm.json > /tmp/input.txt
(while IFS= read -r line; do echo "$line"; sleep 0.08; done < /tmp/input.txt) | java -cp ../.. mnm_vm
# Output: 120

# FizzBuzz (numeric only — PRINT_STR not supported by true interpreter):
python3 ../../mnm_to_stdin.py fizzbuzz.mnm fizzbuzz.mnm.json > /tmp/input.txt
(while IFS= read -r line; do echo "$line"; sleep 0.08; done < /tmp/input.txt) | java -cp ../.. mnm_vm
# Output: 1 2 4 7 8 11 13 14  (Fizz/Buzz strings not printed)
```

## What's in this folder

| MnM source | ArnoldC (compiled) | Description | Output |
|------------|-------------------|-------------|--------|
| `hello_world.mnm` | `mnm_hello_world.arnoldc` | Prints "Hello, world!" | `Hello, world!` (compiled only — uses PRINT_STR) |
| `factorial.mnm` | `mnm_factorial.arnoldc` | Computes 5! | `120` |
| `fizzbuzz.mnm` | `mnm_fizzbuzz.arnoldc` | FizzBuzz 1–15 | Full output (compiled), numeric only (true interpreter) |

The compiled `.arnoldc` files are generated by `generate_mnm_interpreter.py` (the compiler approach). They can also be run via the true interpreter `mnm_vm.arnoldc` for programs within its limits (100 instructions, 50 variables, no strings).
